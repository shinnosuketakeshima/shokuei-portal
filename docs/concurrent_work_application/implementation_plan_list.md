# 兼務届け申請一覧機能の実装計画

提出された兼務届けのデータを一覧で確認できる画面を新しく作成します。

## Proposed Changes

### 1. 新規画面コンポーネントの作成 (`ConcurrentWorksList`)
`src/App.jsx` に新しいコンポーネントを追加します。
- Firestoreの `concurrent_works` コレクションからデータを取得（`useEffect`と`getDocs`を使用）。
- データをテーブル形式で表示します。
- **表示項目（予定）:**
  - 申請日（Firestoreの `createdAt` から変換）
  - 申請教員（氏名）
  - 兼務先（相手先大学等名）
  - 期間（開始日 〜 終了日）

### 2. ルーティングの追加
- `App.jsx` の `<Routes>` 内に `/applications-list` のルートを追加し、上記の `ConcurrentWorksList` を割り当てます。

### 3. サイドバーメニューの更新
- 左側のナビゲーションメニューに、新しく **「申請一覧（兼務履歴）」** へのリンクアイコンを追加し、いつでも一覧画面にアクセスできるようにします。

## Open Questions

> [!NOTE]
> 現在の仕様では、まだ「誰がログインしているか（教員ごとの認証）」の機能が入っていないため、**「全員が提出したすべての兼務届け」が一覧に表示される形**になります。
>
> （後日、ログイン機能を追加した際に「自分の申請だけを表示する」といった権限設定を行う形になります）
> 
> まずは「全員の申請が見える状態」での一覧画面として実装を進めてよろしいでしょうか？

## Verification Plan

1. 実装後、ローカルサーバー（`npm run dev`）でサイドバーから「申請一覧」をクリックする。
2. これまでにテスト送信した申請データがテーブルに正しく表示されるか確認する。
3. 表示に問題がなければ、Firebaseへ本番デプロイを行う。
