# 学外非常勤講師兼務承認申請フォームの改修計画

既存のWordファイル（`26.学外非常勤講師兼務承認申請書（R3.1）.docx`）を解析し、申請に必要な項目を抽出しました。まずはこの内容に合わせて、ポータルの申請フォームを拡張します。

## User Review Required

> [!IMPORTANT]
> フォームの入力項目が大幅に増えます。申請のたびに全ての項目を入力するのは手間になる可能性があるため、**「所属・職名・氏名」などの基本情報は、将来的にユーザーのログイン情報から自動入力できるような設計**を想定していますが、今回の改修では手動入力として実装してよろしいでしょうか？

## Open Questions

> [!NOTE]
> 1. Word出力のためのテンプレート化について：
> 後続のステップで、入力データを元にWordファイル（`.docx`）を自動生成してダウンロードする機能を実装します。これには `docxtemplater` などのライブラリを使用するのが適していますが、そのためには現在のWordファイルをベースにした**テンプレートファイル（ `{{氏名}}` などの変数を埋め込んだdocxファイル ）**をプロジェクト内に作成・配置する必要があります。この作業も併せて進めてよろしいでしょうか？
> 
> 2. 「はい・いいえ」の選択について：
> 誓約事項（「同等科目の依頼がされていないか」「勤務に支障がないか」）に関して、「いいえ」が選択された場合の制御（エラーにして送信できないようにするか、そのまま送信させるか）にご希望はありますか？（標準ではそのまま送信できるようにします）

## Proposed Changes

### UI Components (`src/App.jsx`)

`ConcurrentWorkForm` コンポーネントを大幅に拡張し、以下のセクションに分けてフォーム項目を配置します。

#### 1. 申請者情報
- 所属 (Text)
- 職名 (Text)
- 氏名 (Text)

#### 2. 兼務先・授業情報
- 相手先大学等名 (Text)
- 授業名 (Text)
- 担当時間 (Radio: 前期 / 後期 / 通年 / 集中講義)
- 担当期間 (Date - Date)
- 曜日・時限 (Text)
- 単位数 (Number)

#### 3. 本学との関係及び支障の有無
- 担当授業と本学における担当授業及び専門分野との関係コメント (Textarea)
- 同等科目の依頼有無確認 (Radio: はい / いいえ)
- 本学の勤務・授業等の遂行に支障が生じないことの説明コメント (Textarea)
- 勤務に支障が無いことの確認 (Radio: はい / いいえ)

### Database (`src/App.jsx` - Firestore submission)

Firestoreへ保存する際のデータ構造を、上記の全項目を保持する形式（JSONオブジェクト）にアップデートします。

### [MODIFY] src/App.jsx

フォームの状態管理 (`useState` の `formData`) を拡張し、入力UIを構築、Firestoreの `addDoc` 処理に入力データを渡すよう改修します。

4. Firebaseコンソール（またはローカルのFirestoreデータ）で、入力したデータが正しく保存されているか確認する。
